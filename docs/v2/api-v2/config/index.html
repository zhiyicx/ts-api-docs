# 启动信息的移动端处理方式

移动端在版本发布支出将部分配置信息的默认配置打包写入本地.应用使用过程中更新启动信息,优先使用服务器提供的最新的信息.

由于移动端是在APP启动异步获取的配置信息，在管理后台修改配置后，需要重启APP已获取最新的配置信息

# 列出所有启动者配置

GET api/v2/bootstrappers
